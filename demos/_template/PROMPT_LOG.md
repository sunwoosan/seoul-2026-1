# PROMPT_LOG.md

GitHub Copilot Chat에 입력한 프롬프트를 시간 순서대로 기록합니다.

> **왜?** 좋은 프롬프트의 감각은 시행착오를 봐야 늡니다.
> 코드를 베끼는 게 아니라 프롬프트를 배우는 과정입니다.

---

## #1 — 프로젝트 초기 골격

> **Prompt**
> ## 프롬프트 기록 최종본

### 1. 사이드바 CSV 업로더 추가
**프롬프트**
```text
학급 평가 결과 csv를 업로드하면 성적 분포를 막대그래프로 시각화해주고 점수대별 분포를 나타내어 주는 도구를 만드려고 합니다. 우선 학급 평가 결과 CSV를 업로드할 수 있는 사이드바 CSV 업로더를 만들어주세요
```

**Copilot 반영 결과**
- 사이드바에 CSV 파일 업로더를 추가했다.
- CSV 파일이 업로드되지 않았을 때 안내 문구를 보여주도록 구성했다.
- 업로드 후 CSV를 읽어 DataFrame으로 불러오도록 설정했다.

---

### 2. 표 + 응답자 수 메트릭 카드 추가
**프롬프트**
```text
이제 표 + 응답자 수 메트릭 카드를 추가하고 싶습니다. 해주세요.
```

**Copilot 반영 결과**
- 업로드된 CSV 데이터를 표로 출력하도록 추가했다.
- 표 위에 `st.metric()`을 사용해 응답자 수를 큰 글씨로 표시하도록 구성했다.

---

### 3. 기능 2 추가 — 문항별 응답 분포
**프롬프트**
```text
이제 두 번째 기능을 추가하고 싶습니다. 기능 동작 개요는 다음과 같습니다.

사용자 동작 : 문항 이름을 드롭다운에서 선택한다
화면 결과 : 선택한 문항의 응답 분포가 막대그래프로 나타난다
```

**Copilot 반영 결과**
- 드롭다운(`st.selectbox`)으로 문항 이름을 선택할 수 있게 했다.
- 선택한 문항의 응답 개수를 집계하도록 했다.
- 응답 분포를 막대그래프로 표시하도록 구현했다.

---

### 4. 기능 3 추가 — 전체 요약 보기
**프롬프트**
```text
세 번째 기능을 추가하고 싶습니다. 세 번째 기능의 작동 개요는 다음과 같습니다.

사용자 동작 : "전체 요약 보기"버튼을 누른다
화면 결과 : 모든 문항이 점수대 별로 재배치 된다.
```

**Copilot 반영 결과**
- `전체 요약 보기` 버튼을 추가했다.
- 숫자형 문항을 모아 점수대(0~59, 60~69, 70~79, 80~89, 90~100)별로 재분류하도록 했다.
- 결과를 표와 그래프로 확인할 수 있게 했다.

---

### 5. 프로젝트 목적 및 활용 의도 반영
**프롬프트**
```text
수행 평가, 단원 평가 등의 평가 결과를 집계해서 그래프를 통해 도식화해서 보면 표를 통해 확인하는 것보다 자료를 신속하게 파악하는 것이 가능해진다고 생각합니다. 또한 이 자료는 학생들에게 수준별 학습지를 제공하는데 용이한 자료가 될 것입니다. 별다른 추가는 안하셔도 되는데 참고만 해주세요.
```

**Copilot 반영 결과**
- 앱 소개 문구와 프로젝트 설명에 반영했다.
- 평가 결과를 빠르게 파악하고 수준별 학습지 제공에 활용할 수 있다는 목적을 정리했다.

---

### 6. 범위 제외(Out of Scope) 확정
**프롬프트**
```text
이번에 만들지 않을 것들도 말씀드리겠습니다.(범위 제외/OUT OF SCOPE)

☑ 로그인 / 회원가입 기능
☑ 데이터 영구 저장 (DB, 파일 저장)
☑ 외부 AI API 호출 (ChatGPT, Claude 등)
☑ 이메일 발송, 알림
☑ 모바일 반응형 디자인 (웹만 지원)
```

**Copilot 반영 결과**
- 프로젝트 범위를 Streamlit 기반의 단순 분석 웹앱으로 제한했다.
- 로그인, DB 저장, 외부 API, 이메일, 모바일 대응은 제외 범위로 정리했다.

---

### 7. 개인정보 / 보안 점검
**프롬프트**
```text
다음은 개인정보/보안점검 입니다.
☑ 샘플 데이터에 실제 학생 이름 / 정보가 포함되지 않는다
☑ API 키 / 비밀번호를 app.py에 직접 쓰지 않는다
☑ 공개 리포(Public)여도 무방한 데이터만 사용한다
☑ .gitignore에 민감 파일이 포함되어 있다
위 사항들을 점검해주세요
```

**Copilot 반영 결과**
- `sample_data.csv`에 실제 학생 개인정보가 없는지 확인했다.
- `app.py`에 API 키나 비밀번호가 직접 작성되지 않았는지 점검했다.
- 공개 저장소에 올려도 되는 데이터만 사용 중인지 검토했다.
- `.gitignore`에 `.env`, `.streamlit/secrets.toml`, `*.csv` 등의 민감 파일 제외 규칙이 포함되어 있는지 확인했다.
